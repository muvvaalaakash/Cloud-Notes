
## UDR
User-Defined Routes (UDRs) in Azure allow you to override default system routing by creating custom, static routes for traffic within virtual networks. By creating a route table and associating it with subnets, you can direct traffic through network virtual appliances (NVAs) (firewalls, routers) to enhance security and network control



## What is a VM Scale Set?

A **Virtual Machine Scale Set (VMSS)** is a cloud feature (commonly in Microsoft Azure) that lets you deploy and manage a **group of identical virtual machines (VMs)** as a single resource.

Instead of managing individual VMs one by one, a scale set:
- Creates multiple VMs from the same image
- Automatically distributes them across fault domains
- Integrates with a load balancer
- Allows automatic scaling (increase/decrease VM count)

### Key Features
- ✅ Centralized management
- ✅ High availability
- ✅ Load balancing integration
- ✅ Automatic scaling support
- ✅ Rolling upgrades

---

## What is Auto Scaling?

**Auto scaling** is the ability to automatically increase or decrease the number of running VMs based on demand or predefined rules.

It ensures:
- Applications handle high traffic
- Costs are optimized during low usage
- Resources scale without manual intervention

---

## How VM Scale Set + Auto Scaling Work Together

A VM Scale Set provides the infrastructure group of VMs.

Auto scaling defines the rules that control:
- When to add VMs (scale out)
- When to remove VMs (scale in)

---

## Example Scenario

Imagine you run an e-commerce website:

### Normal Day
- 2 VMs handle regular traffic.

### Big Sale Event
- CPU usage rises above 75%.
- Auto scaling rule triggers.
- VMSS increases to 6 VMs automatically.

### After Sale Ends
- CPU drops below 30%.
- VMSS scales back to 2 VMs.
- You save costs.

---

## Types of Auto Scaling Rules

1. **Metric-based scaling**
   - CPU usage
   - Memory usage
   - Request count
   - Queue length

2. **Schedule-based scaling**
   - Scale up at 9 AM
   - Scale down at 10 PM

## Important Terms

- **Scale Out** → Add more VMs  
- **Scale In** → Remove VMs  
- **Horizontal Scaling** → Add/remove machines  
- **Vertical Scaling** → Increase machine size (CPU/RAM)

---

## Simple Difference

| VM Scale Set | Auto Scaling |
|--------------|--------------|
| Group of identical VMs | Logic that changes number of VMs |
| Infrastructure | Policy/Rule |
| Handles deployment | Handles scaling decisions |
